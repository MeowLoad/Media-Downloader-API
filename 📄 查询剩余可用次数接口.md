# 字幕提取接口文档 📄

> API 基于 REST 架构设计。API 具有结构清晰的面向资源的 URL，接收 JSON 格式的请求体，返回 JSON 格式的响应，并使用标准的 HTTP 响应状态码。

该接口用于查询账户内剩余可用次数 （系统会在剩余可用次数低于 10000、2000、300、100、0 这几个节点给您发送余量预警邮件、短信，您可通过该接口实现其他规则的余量预警）

### 请求参数

**接口地址:** `https://api.meowload.net/openapi/available-credits`

**请求方式:** `GET`

**请求头(Header)**

| 请求头            | 说明                                                                            |
| :---------------- | :------------------------------------------------------------------------------ |
| `x-api-key`       | API-Key，在[开发者接口管理](https://www.henghengmao.com/user/developer)页面获取 |
| `accept-language` | 错误返回时 message 的语言，默认`en`，支持`zh`、`en`、`ja`、`es`、`de`等         |

### 🟢 成功返回数据示例

> HTTP 状态码为 200

```json
{
  "availableCredits": 282539
}
```

**返回结果说明**

| 字段               | 说明         | 是否一定有值 |
| ------------------ | ------------ | ------------ |
| `availableCredits` | 剩余可用次数 | ✨ 一定有    |

### 🔴 失败返回示例

> HTTP 状态码非 200，比如 400、422、401、402、500 等

```
{
    "message": "API Key 错误，请检查 API Key 是否正确"
}
```

### 响应 HTTP 状态码说明

| HTTP 状态码 | 说明     | 返回内容示例                                 |
| :---------- | :------- | :------------------------------------------- |
| 200         | 成功     | 参考上述成功返回数据示例                     |
| 401         | 鉴权失败 | 请检查 API-Key 是否正确                      |
| 500         | 未知错误 | 该错误一般不会遇到，如果遇到，请联系技术支持 |
